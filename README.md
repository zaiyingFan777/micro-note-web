### commit规范
- feat：新功能
- update：更新某功能
- fix：修补某功能的bug
- refactor：重构某个功能
- optimize: 优化构建工具或运行时性能
- style：仅样式改动
- docs：仅文档新增/改动
- chore：构建过程或辅助工具的变动

```bash
# 示例
git commit -m 'feat: init'
```
